Outliers and Impact on Machine Learning¶

Which Machine Learning Models are sensitive to outliers i.e getting impacted by outliers?¶

  1. Naive Bayes Classifier ---- Not Sensitive to Outliers
  2. SVM ----------------------- Not Sensitive to Outliers
  3. Linear Regression --------- Sensitive to Outliers
  4. Logistic Regression ------- Sensitive
  5. Decision Tree Regressor or Classifier ---- Not Sensitive
  6. Ensemble (Random Forest, XGBoost, Gradient Boosting) ---- Not Sensitive
  7. KNN ----------------------- Not Sensitive
  8. Kmeans -------------------- Sensitive
  9. Hierarchial ---------------- Sensitive
  10. PCA/LDA ------------------ Sensitive (very sensitive so remove the outliers)
  11. Neural Networks ---------- Sensitive
  12. DBSCAN ------------------- Sensitive

Learning the maths of machine learning algorithms will help you understand them. The Kmeans for e.g. uses Euclidean distance so it will be sensitive to the outliers

All unsupervised learning will be Sensitive to Outliers

If you need say to do a credit card fraud where the outliers are very important and should not be removed, use the algorithm that is not sensitive to outliers

Assuming Age follows a Gaussian/Normal Distribution, we will calculate the boundaries which differentiates the outliers.¶

The mean, 3-Std Dev is considered.

99% of data is present within the 3-Std Dev

In [9]:
### make upper boundary
upper_boundary = df['Age'].mean() + 3 * df['Age'].std()  # 3 *: represents 3 Std Dev.
lower_boundary = df['Age'].mean() - 3 * df['Age'].std()  # 3 *: represents 3 Std Dev.
print(lower_boundary), print(upper_boundary), print(df['Age'].mean())

Below is for demonstration purposes only because it's used more for skewed data. The 3 standard deviation to the right and 3 standard deviation to the left we used above it what should be used if you have a normal/gaussian distribution. ### Let's compute the Interquantile range to calculate the boundaries IQR = df.Age.quantile(0.75) - df.Age.quantile(0.25) IQR### calculating the lower and upper bridge lower_bridge = df['Age'].quantile(0.25) - (IQR*1.5) upper_bridge = df['Age'].quantile(0.75) + (IQR*1.5) print(lower_bridge), print(upper_bridge)It means that Age value above 64 we consider as an outlier We will not consider the -6.6875 because Age cannot be a negative value. To calculate the extreme boundaries we multiply by 3 i.e. 3 Std Dev.### Extreme Outliers lower_bridge = df['Age'].quantile(0.25) - (IQR*3) upper_bridge = df['Age'].quantile(0.75) + (IQR*3) print(lower_bridge), print(upper_bridge)This means that values above 91 are to be considered as outliers. However, our maximum value for Age is 80.

If it's left skewed, you take the left value of the extreme outlier result.

Removing Outliers¶

  • If the dataset does not have a lot of observations, you can replace them with a value instead of removing them.
  • whereever you have values greater than 73, replace those outliers with the upper_boundary value of 73
  • For Fare, replace the outliers with upper boundary value of 100
